# 个人博客网站设计文档

## 目标

基于 `Astro` 构建一个适合部署到 `Vercel` 的个人博客网站，整体定位为“现代极简”的内容型博客，强调阅读体验、版式节奏、清晰的信息架构与低维护成本。

## 项目范围

首期版本包含以下页面与能力：

- 首页
- 文章列表页
- 文章详情页
- 关于页
- 标签总览页
- 单标签文章列表页

首期版本不包含以下能力：

- 评论系统
- 站内搜索
- 登录后台
- 数据库
- 订阅系统
- 复杂动画或重交互模块

## 用户目标

- 让访客快速理解作者是谁、在写什么
- 让读者能够高效浏览最新文章与精选文章
- 让长文阅读保持舒适、专注、稳定
- 让维护者通过 Markdown 即可新增和管理文章
- 让站点可以低成本部署到 `Vercel`

## 技术方案

### 核心栈

- `Astro`
- `TypeScript`
- `Markdown` 内容文件
- 原生 `CSS` 或项目内简洁样式方案

### 部署策略

- 采用静态构建输出
- 优先兼容 `Vercel` 静态部署流程
- 不依赖数据库与服务端运行时

### 内容管理

文章通过本地 Markdown 文件维护，每篇文章使用 frontmatter 管理元信息。建议字段如下：

```md
---
title: 文章标题
description: 文章摘要
pubDate: 2026-04-13
tags:
  - 标签一
  - 标签二
featured: true
---
```

站点在构建时自动生成：

- 首页精选文章
- 最新文章列表
- 全部文章列表
- 标签总览
- 单标签归档页

## 信息架构

### 首页

首页承担“建立印象 + 提供阅读入口”的双重作用，结构如下：

1. 顶部导航
2. 首屏 Hero 区
3. 精选文章区
4. 最新文章列表区
5. 页脚

#### 顶部导航

保留最少入口，降低认知负担：

- 文章
- 标签
- 关于

#### 首屏 Hero 区

展示博客名称、作者一句话介绍、简短写作宣言，并提供通往文章区的阅读入口。视觉上以大字号标题和留白建立识别度，不使用冗余插画和装饰性组件。

#### 精选文章区

展示 2 到 3 篇精选文章。样式采用编辑推荐式排版，强调标题与摘要，不做密集卡片堆叠。

#### 最新文章列表区

按时间倒序展示文章，信息包括：

- 日期
- 标题
- 摘要
- 标签

该区域是首页的主要使用场景，需优先保证可读性与快速扫描能力。

### 文章列表页

集中展示全部文章，使用清晰、规整的纵向列表布局。用户可以快速按时间浏览内容，不引入复杂筛选器。

### 文章详情页

文章详情页以阅读体验为核心，页面应包含：

- 标题
- 发布日期
- 标签
- 正文内容
- 上下篇导航或回到列表入口

重点优化项：

- 正文最大宽度控制
- 标题、段落、引用、代码块、列表的层级清晰
- 行高与段间距舒适
- 移动端保持稳定阅读体验

### 关于页

展示作者简介、写作方向与基础联系方式。视觉风格与正文保持一致，避免做成另一套页面语言。

### 标签页

标签能力包含两层：

- 标签总览页：展示所有标签及对应文章数量
- 单标签页：展示该标签下的文章列表

标签是对文章列表的补充导航，不喧宾夺主。

## 视觉设计

### 风格关键词

- 现代极简
- 黑白灰
- 内容优先
- 大留白
- 强排版
- 弱装饰

### 视觉原则

- 用字号、粗细、间距建立层级，而不是依赖大量边框和阴影
- 页面以浅色背景为主，避免花哨纹理与高饱和配色
- 交互反馈克制明确，保留必要的悬停与焦点状态
- 重点通过节奏感而非装饰物体现设计感

### 排版策略

- 首页标题采用更强识别度的字重与字号
- 列表信息遵循固定节奏，保证扫描效率
- 文章正文的宽度、行高和段落节奏优先于“塞满屏幕”
- 移动端按同样原则缩放，不牺牲可读性

## 组件设计

建议拆分为以下基础组件：

- `Layout`：全局头部、页脚、SEO 基础结构
- `Header`：站点导航
- `Hero`：首页首屏介绍
- `FeaturedPosts`：精选文章模块
- `PostList`：文章列表模块
- `TagList`：标签列表模块
- `PostMeta`：文章日期与标签信息
- `Footer`：页脚信息

组件边界要求：

- 每个组件只处理单一职责
- 列表渲染逻辑与展示样式分离
- 页面只负责拼装内容，不承担过多数据整理逻辑

## 数据流

整体数据流采用构建时读取内容的方式：

1. Astro 从内容目录读取文章 Markdown
2. 根据 frontmatter 生成文章元数据
3. 页面构建时筛选精选文章、最新文章、标签映射
4. 生成静态页面并输出用于 `Vercel` 部署

该流程避免运行时请求数据，降低复杂度与部署成本。

## 错误处理与边界

- 缺失文章字段时，在开发环境提供清晰报错
- 标签为空时允许正常渲染，但不展示无意义标签块
- 无精选文章时，首页精选区应优雅降级为普通文章推荐或隐藏
- 列表为空时，页面需提供明确的空状态说明

## 示例内容策略

首期实现中预置若干示例文章，用于展示：

- 首页精选区效果
- 列表排版效果
- 标签归档效果
- 正文元素样式效果

示例文章需要覆盖普通段落、列表、引用、代码块等常见正文元素。

## 测试策略

首期至少覆盖以下验证范围：

- 首页、文章列表、文章详情、关于页、标签页可正常访问
- Markdown 文章能正确生成详情页
- 标签归档生成正确
- 关键页面在桌面端与移动端布局可用
- 构建命令可通过，产物可用于 `Vercel` 部署

如果项目集成自动化测试，则优先覆盖：

- 内容集合 schema 校验
- 关键页面渲染
- 标签生成逻辑

## 实施边界

本次实施目标是完成一个可部署、可继续写作的个人博客基础版本，不扩展到完整内容平台。后续如果要增加搜索、评论、订阅、项目页等能力，应在该基础上单独规划。

## 验收标准

- 站点可在本地运行并成功构建
- 站点可部署到 `Vercel`
- 首页、文章列表、文章详情、关于页、标签页全部完成
- 视觉风格符合“现代极简、内容优先”的既定方向
- 新增文章仅需添加 Markdown 文件，无需修改多处代码
